Important FAQ

What is the Document Object Model (DOM)

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree of objects, making it possible for programming languages (like JavaScript) to interact with the document.

How do you select an element from the DOM?

To select an element from the DOM, you can use various JavaScript methods depending on what you're looking for (like an element's tag, ID, class, or other attributes). common methods are:
Use document.getElementById() to get an element with a specific id attribute.
Use document.getElementsByClassName() to get all elements with a specific class. This returns an HTMLCollection (like an array, but not exactly).
Use document.getElementsByTagName() to get all elements with a specific tag name (e.g. <p>,<div>) .
document.querySelector(): Selects the first matching element.
document.querySelectorAll(): Selects all matching elements. It returns a NodeList, which can be iterated over.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in DOM manipulation where you take advantage of event bubbling to handle events for multiple child elements by attaching a single event listener to their parent. This eliminates the need to add individual event listeners to each child element.

How do you manipulate an element's attributes and styles using the DOM?

Manipulating Attributes:
1.Use the getAttribute() method to retrieve the value of an attribute.
2.Use the setAttribute() method to modify or add an attribute.
3. Use the removeAttribute() method to remove an attribute. 4. For standard attributes (e.g. href,target), you can manipulate them directly as properties.

Manipulating Styles:
1.The style property allows you to set individual CSS properties. Use camelCase for multi-word properties. (e.g. div.style.backgroundColor = "blue";)
2.For computed styles (e.g., styles applied via CSS files or inherited), use getComputedStyle()
3. Using classes (e.g. node.classList.add,remove,toggle,contains) is a better approach for managing styles, as it keeps your CSS centralized.
